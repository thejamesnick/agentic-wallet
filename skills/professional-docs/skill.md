# Professional Documentation Skill

This skill transforms technical documentation to achieve Apple-grade professional quality—clear, concise, minimalist, and free of AI-generated characteristics.

## When to Use This Skill

Use this skill when you want to:
- Improve README.md, API documentation, or technical guides
- Remove AI-like verbosity and generic phrasing
- Achieve consistent, professional documentation style
- Ensure documentation is user-focused and action-oriented

## How It Works

The skill analyzes your documentation and provides specific recommendations for:

1. **Conciseness**: Reducing word count while preserving meaning
2. **Precision**: Replacing vague terms with specific technical details
3. **Structure**: Improving information hierarchy and flow
4. **Tone**: Ensuring professional, confident voice without hype
5. **Format**: Consistent code blocks, headings, and lists
6. **Content**: Identifying missing essential information

## Usage

Invoke this skill on your documentation files:
```
/professional-docs README.md
/professional-docs docs/API.md
/professional-docs ./docs/
```

The skill will output specific, actionable suggestions for improvement.

## Philosophy

Inspired by Apple's documentation principles:
- Clarity over cleverness
- Brevity over completeness
- Precision over pretension
- Utility over utility-belt thinking

Good documentation answers: What is this? How do I use it? Why should I care?—in that order.

## Examples of Improvements

**Before (AI-like)**:
"In today's fast-paced technological landscape, our innovative solution leverages cutting-edge blockchain technology to revolutionize the way developers interact with decentralized applications, providing unparalleled flexibility and scalability for modern web3 implementations."

**After (Professional)**:
"This Solana wallet enables AI agents to execute transactions programmatically. Developers integrate it via the @pocketagent/paw npm package."

**Before**:
"We are excited to announce our amazing new feature that will totally transform your experience and make everything better than ever before!"

**After**:
"Intent-based commands allow agents to execute trades using natural language: `paw buy --agent-id trader --token SOL --budget 1.0`"

## Implementation Notes

This skill focuses on:
- Active voice over passive
- Specific examples over general descriptions
- Consistent terminology throughout
- Progressive disclosure (simple to complex)
- Anticipating user questions
- Eliminating marketing fluff
- Technical accuracy as the priority

The goal is documentation that feels inevitable—like it was always obvious how things should work.
